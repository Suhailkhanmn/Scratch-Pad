# PRD

## Summary
Scratch Pad shows the primary result on one screen and lets the user enter the main input on one screen.

## Scope
- Capture the main user input on one screen
- Show the primary result on one screen
- Save project data locally

## Acceptance
- A user can enter the main input on one screen and continue the primary flow.
- The app shows the primary result on one screen in the main flow.
- Saved project data is still available after reload without any remote service.

## Non-goals
- No secondary workflows outside the primary use case
- No expansion beyond the main happy path
